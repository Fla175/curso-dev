Guia de uso do cURL para testes de API

A sintaxe básica do cURL para fazer uma requisição HTTP é: 

curl [opções] [URL]

1. Requisição GET

O método GET é usado para solicitar dados de um recurso específico. É o método padrão do cURL, então você não precisa especificar o verbo HTTP explicitamente.

Exemplo:

curl https://api.exemplo.com/usuarios/123

Com opções adicionais:

  • -i ou --include: Inclui os cabeçalhos da resposta na saída.
  • -v ou --verbose: Mostra detalhes da requisição e da resposta, útil para depuração.
curl -i -v https://api.exemplo.com/usuarios/123

2. Requisição POST

O método POST é usado para enviar dados a um servidor para criar um novo recurso. Para enviar dados, você pode usar a opção -d ou --data.

Exemplo:

curl -X POST \
  -H "Content-Type: application/json" \
  -d '{ "nome": "Fulano de Tal", "email": "fulano@exemplo.com" }' \
  https://api.exemplo.com/usuarios

Explicação dos parâmetros:

  • -X POST: Especifica o verbo HTTP como POST.
  • -H "Content-Type: application/json": Adiciona um cabeçalho Content-Type para informar ao servidor que o corpo da requisição está no formato JSON.
  • -d ou --data: Envia os dados do corpo da requisição. Se os dados forem um arquivo, você pode usar @nome-do-arquivo.json.

3. Requisição PUT

O método PUT é usado para atualizar um recurso existente. A sintaxe é muito similar ao POST, mas você deve especificar o ID do recurso que será atualizado na URL.

Exemplo:

curl -X PUT \
  -H "Content-Type: application/json" \
  -d '{ "email": "novo_email@exemplo.com" }' \
  https://api.exemplo.com/usuarios/123

4. Requisição DELETE

O método DELETE é usado para remover um recurso específico. Normalmente não há corpo na requisição DELETE.

Exemplo:

curl -X DELETE https://api.exemplo.com/usuarios/123

Resumo dos principais comandos e opções:

Comando Descrição
curl [URL] Requisição GET simples.
-X [VERBO] Define o verbo HTTP (GET, POST, PUT, DELETE).
-d [DADOS] Envia dados no corpo da requisição (para POST, PUT).
-H "[CABEÇALHO]" Adiciona um cabeçalho à requisição.
-i Exibe os cabeçalhos da resposta.
-v Modo detalhado (verbose) para depuração.
-o [ARQUIVO] Salva a resposta em um arquivo.
-u [USUARIO:SENHA] Adiciona autenticação básica.
